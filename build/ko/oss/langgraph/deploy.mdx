LangGraph Platform은 에이전트를 프로덕션 시스템으로 전환하는 가장 빠른 방법입니다. 기존 호스팅 플랫폼은 상태 비저장, 단기 실행 웹 앱을 위해 구축되었지만, LangGraph는 **상태 기반의 장기 실행 에이전트를 위해 특별히 설계**되어 있어 저장소에서 안정적인 클라우드 배포까지 몇 분 만에 완료할 수 있습니다.

## 사전 요구사항

시작하기 전에 다음이 필요합니다:

* [GitHub 계정](https://github.com/)
* [LangSmith 계정](https://smith.langchain.com/) (무료 가입 가능)

## 에이전트 배포하기

### 1. GitHub에 저장소 생성하기

애플리케이션의 코드는 LangGraph Platform에 배포하기 위해 GitHub 저장소에 있어야 합니다. 공개 및 비공개 저장소가 모두 지원됩니다. 이 빠른 시작에서는 먼저 [로컬 서버 설정 가이드](/oss/python/langgraph/studio#setup-local-langgraph-server)를 따라 앱이 LangGraph와 호환되는지 확인한 다음, 코드를 저장소에 푸시하세요.

### 2. LangGraph Platform에 배포하기

<Steps>
<Step title="LangSmith Deployments로 이동">
    [LangSmith](https://smith.langchain.com/)에 로그인하세요. 왼쪽 사이드바에서 **Deployments**를 선택하세요.
</Step>
<Step title="새 배포 생성">
    **+ New Deployment** 버튼을 클릭하세요. 필수 필드를 입력할 수 있는 패널이 열립니다.
</Step>
<Step title="저장소 연결">
    처음 사용하는 경우이거나 이전에 연결하지 않은 비공개 저장소를 추가하는 경우, **Add new account** 버튼을 클릭하고 안내에 따라 GitHub 계정을 연결하세요.
</Step>
<Step title="저장소 배포">
    애플리케이션의 저장소를 선택하세요. **Submit**을 클릭하여 배포하세요. 완료까지 약 15분 정도 소요될 수 있습니다. **Deployment details** 보기에서 상태를 확인할 수 있습니다.
</Step>
</Steps>

### 3. LangGraph Studio에서 애플리케이션 테스트하기

애플리케이션이 배포되면:

1. 방금 생성한 배포를 선택하여 자세한 정보를 확인하세요.
2. 오른쪽 상단의 **LangGraph Studio** 버튼을 클릭하세요. LangGraph Studio가 열리면서 그래프를 표시합니다.

### 4. 배포의 API URL 가져오기

1. LangGraph의 **Deployment details** 보기에서 **API URL**을 클릭하여 클립보드에 복사하세요.
2. `URL`을 클릭하여 클립보드에 복사하세요.

### 5. API 테스트하기

이제 API를 테스트할 수 있습니다:

<Tabs>
<Tab title="Python">
1. LangGraph Python을 설치하세요:
```shell
pip install langgraph-sdk
```
2. 에이전트에 메시지를 보내세요:
```python
from langgraph_sdk import get_sync_client # 또는 비동기의 경우 get_client

client = get_sync_client(url="your-deployment-url", api_key="your-langsmith-api-key")

for chunk in client.runs.stream(
    None,    # 스레드 없는 실행
    "agent", # 에이전트 이름. langgraph.json에 정의됨.
    input={
        "messages": [{
            "role": "human",
            "content": "What is LangGraph?",
        }],
    },
    stream_mode="updates",
):
    print(f"Receiving new event of type: {chunk.event}...")
    print(chunk.data)
    print("\n\n")
```
</Tab>


<Tab title="Rest API">
```bash
curl -s --request POST \
    --url <DEPLOYMENT_URL>/runs/stream \
    --header 'Content-Type: application/json' \
    --header "X-Api-Key: <LANGSMITH API KEY> \
    --data "{
        \"assistant_id\": \"agent\", `# 에이전트 이름. langgraph.json에 정의됨.`
        \"input\": {
            \"messages\": [
                {
                    \"role\": \"human\",
                    \"content\": \"What is LangGraph?\"
                }
            ]
        },
        \"stream_mode\": \"updates\"
    }"
```
</Tab>
</Tabs>

<Tip>
LangGraph Platform은 자체 호스팅 및 하이브리드를 포함한 추가 배포 옵션을 제공합니다. 자세한 내용은 [배포 옵션](/langgraph-platform/deployment-options)을 참조하세요.
</Tip>
